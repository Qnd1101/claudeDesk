## 3. 화면 전이도 (상태 → 키 → 상태)

```text
                 ┌──────────────────────────── q/Esc ──────────────┐
                 ▼                                                 │
            [Main List] ──Enter──▶ resume(핸드오프, 앱 종료) ──▶ claude 진입
              │  ▲
   / ─────────┘  │ Esc
   ▼             │
 [Search] ──Enter(매칭)──▶ resume
   │
   │  (Main List에서)
   ├─ s ─▶ [Main List/정렬변경] (인플레이스)
   ├─ g ─▶ [Main List/그룹토글] (인플레이스)
   ├─ p ─▶ [Preview 분할] ◀─ p ─┐
   ├─ Space ─▶ 다중선택 토글(인플레이스)
   ├─ Del ─▶ [삭제 확인 모달] ──Enter──▶ 휴지통 이동·복귀 / ──Esc──▶ 복귀
   ├─ R ─▶ [별칭 모달] ──Enter/Esc──▶ 복귀
   ├─ T ─▶ [휴지통] ──(r 복구 / D 영구삭제)──▶  ──Esc──▶ 복귀
   ├─ , ─▶ [설정] ──s 저장/Esc──▶ 복귀
   └─ ? ─▶ [도움말 오버레이] ──Esc/?──▶ 복귀
```

* **모달 규칙:** 모달 활성 시 하위 키 차단, `Esc`는 항상 한 단계 닫기. resume만 앱을 종료(핸드오프).

## 6. 반응형 (좁은 터미널)

| 폭 | 컬럼 정책 |
| :--- | :--- |
| ≥120 | 전체: 마커·제목·프로젝트·수정시각·메시지수 (+미리보기 분할 가능) |
| 80~119 | 프로젝트를 폴더명만(말줄임), 미리보기는 전체화면 토글로 |
| <80 | 제목·수정시각만. 프로젝트/메시지수 숨김. 모달은 폭 맞춤 축소 |
| <40(극협) | 제목 1열 + 상태바 1줄. 경고 "터미널을 넓혀주세요" 옵션 |

* 제목은 `unicode-width`로 grapheme 폭 계산해 안전 말줄임(`…`). 이모지/CJK 폭 2칸 처리(§5.6).

## 7. 컬러/상태 표기 규칙

| 의미 | 색(기본 dark) | 기호(컬러무관) |
| :--- | :--- | :--- |
| 선택 커서 | 반전(invert) | `▸` |
| 활성 세션 | 초록 | `●` + "active" |
| 다중선택 | 청록 | `✓` |
| 삭제 임박(휴지통) | 노랑 | `⚠` |
| 에러/스킵 | 빨강 | `!` + 상태바 카운트 |
| 핀(별칭화면) | 보라 | `*` |

* **테마:** `auto`(터미널 배경 감지)/`dark`/`light`/`mono`. `mono`·`NO_COLOR`에서는 색 제거, **기호만으로 모든 상태 식별 가능해야 함**(접근성 필수, §5.7).

## 8. 접근성 체크리스트

* [ ] 모든 동작 키보드 도달(마우스 불필요).
* [ ] 상태를 색에만 의존하지 않음(기호 병기).
* [ ] `NO_COLOR`/`mono` 정상 동작.
* [ ] 핵심 정보 텍스트 라벨 제공(스크린리더 환경 best-effort).
* [ ] 좁은 폭에서도 크래시 없이 정보 우선순위 축약.
* [ ] 위험 동작(영구삭제)은 2단계 확인.
